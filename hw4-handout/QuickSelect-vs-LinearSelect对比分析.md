# QuickSelect vs LinearSelect 性能对比分析

## 一、实验目的

比较两种经典选择算法的实际性能：

- **QuickSelect**：随机选择 pivot，期望 $O(n)$ 时间复杂度，常数小
- **LinearSelect**（Median of Medians）：通过分组中位数保证 pivot 质量，严格 $O(n)$ 最坏情况，含可调参数 Q

探究不同数据规模、数据分布对两者的影响差异。

---

| 参数 | 取值 |
|:---:|:-----|
| Q 值 | 3, 5, 7, 9, 11, 15, 21, 31, 41, 51, 75, 101 |
| 数据规模 N | 100, 500, 1000, 5000, 10000, 20000 |
| 数据分布 | 随机、顺序、重复（10% 去重值）、近似有序 |
| 试验次数 | 200~500 次/组 |
| 指标 | P90 时间(μs)、平均/最大递归深度、平均/最大调用次数 |

编译：`g++ -std=c++20 -O2 -o median.exe median.cc`

## 三、核心结果：QuickSelect vs LinearSelect

### 3.1 性能总览（N=5000，随机数据）

| 算法 | P90 时间(μs) | 说明 |
|:----|:----------:|:-----|
| **QuickSelect** | **≈ 40** | 随机 pivot，期望 $O(n)$，常数极小 |
| LinearSelect (Q=3) | 311 | 性能最差，递归树爆炸导致常数极大；最坏情况退化为 $O(n \log n)$ |
| LinearSelect (Q=5) | 131 |  |
| LinearSelect (Q=21) | 109 | |
| LinearSelect (Q=51) | 89 | |
| LinearSelect (Q=101) | 69 | 最优，但仍比 QuickSelect 慢 |

$$
\text{QuickSelect 比 LinearSelect(Q=101) 快约 } \frac{69}{40} \approx 1.7\text{ 倍}
$$

$$
\text{QuickSelect 比 LinearSelect(Q=5) 快约 } \frac{131}{40} \approx 3.3\text{ 倍}
$$

**结论**：QuickSelect 在所有 Q 值下均显著快于 LinearSelect，差距随 Q 增大而缩小。

---

### 3.2 不同数据规模下的对比

下列为不同 N 下 LinearSelect（最优 Q 值，P90）与 QuickSelect（P90）的对比。

| N | QuickSelect P90(μs) | LinearSelect 最优 P90(μs) | 比值 |
|:-:|:------------------:|:------------------------:|:---:|
| 5000 | ≈40 | 69 (Q=101) | **1.7×** |
| 10000 | 80 | 149 (Q=101) | **1.9×** |
| 20000 | 163 | 313 (Q=101) | **1.9×** |

从增长趋势看：

- N=5000 → 10000，QuickSelect 增长 2.00×（40→80），LinearSelect 增长 2.16×（69→149）
- N=10000 → 20000，QuickSelect 增长 2.04×（80→163），LinearSelect 增长 2.10×（149→313）

两者均严格呈线性增长。LinearSelect 的常数因子约为 QuickSelect 的 **1.7~3×**（取决于 Q）。

## 四、数据分布对两者的影响

### 4.1 LinearSelect Q=5 下各分布表现（N=5000）

> QuickSelect 不含参数 Q，此处作为全局基准对照组（Baseline）列出，其数据与 Q 无关。

| 数据分布 | Linear P90(μs) | Quick P90(μs) | 比值 (L/Q) |
|:-------:|:-------------:|:-------------:|:---------:|
| 随机 | 134 | 42 | 3.2× |
| 顺序 | 98 | 16 | 6.1× |
| 重复 | 132 | 43 | 3.1× |
| 近似有序 | 75 | 18 | 4.2× |

### 4.2 LinearSelect Q=21 下各分布表现（N=5000）

| 数据分布 | Linear P90(μs) | Quick P90(μs) | 比值 (L/Q) |
|:-------:|:-------------:|:-------------:|:---------:|
| 随机 | 114 | 43 | 2.7× |
| 顺序 | 26 | 12 | 2.2× |
| 重复 | 114 | 42 | 2.7× |
| 近似有序 | 55 | 16 | 3.4× |

### 4.3 分布影响分析

**各分布下的波动幅度（max/min）：**

| 算法 | 4.1 节数据 | 4.2 节数据 |
|:----|:--------:|:--------:|
| QuickSelect（基准） | 43/16 = **2.69×** | 43/12 = **3.58×** |
| LinearSelect | 134/75 = **1.79×** | 114/26 = **4.38×** |

**关键发现：**

1. **顺序数据最快**：原因并非 pivot 质量更高——随机数据下 pivot 期望上同样分割均匀。真正原因是 **CPU 分支预测（Branch Prediction）**对有序数据的比较结果几乎 100% 命中，避免流水线停顿；同时**缓存局部性（Cache Locality）**使 partition 中的内存访问完全连续，大幅减少 cache miss
2. **Q=5 时 LinearSelect 波动更小**（2.69× vs 1.79×），**Q=21 时 LinearSelect 波动更大**（3.58× vs 4.38×）——大 Q 下中位数数组变小，pivot 质量对数据分布更敏感
3. **重复数据对两者耗时与随机数据接近**：三路划分将等值元素排除在递归之外，有效缓解了重复值的不利影响

## 五、递归行为对比

### 5.1 Q 对递归深度的影响

| Q | N=5000 深度 | N=10000 深度 | N=20000 深度 |
|:-:|:----------:|:-----------:|:-----------:|
| 3 | 12.10 | 13.17 | 14.15 |
| 5 | 11.20 | 12.28 | 13.40 |
| 7 | 10.97 | 11.88 | 12.78 |
| 21 | 9.38 | 10.34 | 11.31 |
| 51 | 8.09 | 9.10 | 10.14 |
| 101 | 7.01 | 8.07 | 9.07 |

- 递归深度随 Q 增大**单调递减**：Q=3→101 减少约 40%
- N 翻倍（5000→10000→20000），深度约增加 1

### 5.2 Q 对调用次数的影响

| Q | N=5000 | N=10000 | N=20000 |
|:-:|:------:|:-------:|:-------:|
| 3 | 766.55 | 1184.40 | 2127.89 |
| 5 | 192.13 | 338.21 | 432.47 |
| 7 | 126.69 | 182.13 | 259.19 |
| 21 | 35.23 | 50.18 | 69.91 |
| 51 | 19.15 | 24.26 | 33.76 |
| 101 | 13.00 | 15.13 | 21.13 |

- Q=3 调用次数极大（N=20000 时 >2100 次），因中位数数组规模 n/3，递归树分支众多
- Q=101 调用次数仅 13~21 次，几乎是单链递归
- **Q=3→101 调用次数减少 98%+**

### 5.3 与 QuickSelect 的递归差异

| | QuickSelect | LinearSelect |
|:--|:----------|:------------|
| 递归方式 | 单分支（尾递归） | 双分支（中位数递归 + 选择递归） |
| 期望深度 | $O(\log n)$ | $O(\log n)$ |
| 调用次数 | 期望 $O(\log n)$ | $O(\log_Q n + \log n)$ |
| 最坏深度 | $O(n)$（退化） | $O(\log n)$（Q≥5 时保证；Q=3 退化为 $O(\log n)$） |

QuickSelect 虽然最坏情况可能退化到 $O(n^2)$，但在随机数据下几乎不会发生（概率极低）。LinearSelect 用额外的递归分支换取了确定性保证——但需注意 **Q=3 时此保证失效**，Median of Medians 在 Q=3 时最坏时间复杂度退化为 $O(n \log n)$，因为中位数数组中位数无法保证常数比例的剔除（见报告 1 第 4.4 节）。

## 七、结论

### 7.1 QuickSelect vs LinearSelect

| 维度 | QuickSelect | LinearSelect |
|:----|:-----------|:------------|
| **实际速度** |  快 1.7~6.1× |  较慢 |
| **理论保证** | 期望 $O(n)$ | Q≥5 时严格 $O(n)$；Q=3 退化为 $O(n \log n)$ |
| **最坏情况** | $O(n^2)$（罕见） | $O(n)$ |
| **分布敏感性** | 波动 2.7~3.6× | 波动 1.8~4.4× |
| **实现复杂度** | 简单 | 较复杂（分组+双递归） |

### 7.2 建议

1. **实际应用中优先使用 QuickSelect**：随机 pivot 在实际数据上几乎不会退化，且常数小得多
2. **关键任务场景选 LinearSelect（Q≥5）**：在实时系统、安全攸关系统或对最坏运行时间有严格约束的场景下，QuickSelect 的 $O(n^2)$ 退化风险不可接受。LinearSelect 提供**确定性的 $O(n)$ 保证**，消灭了性能退化风险。选择 **Q=51~101** 可在保证与性能间取得最佳平衡
3. **Q 越大越好**（在合理范围内）：Q=101 比经典 Q=5 快约 1.9 倍，深度和调用次数均大幅降低
4. **数据已有序时两种算法都快**：受益于 CPU 分支预测和缓存局部性，partition 操作效率大幅提升，与 pivot 质量无关
5. **三路划分有效缓解重复值问题**：重复数据与随机数据耗时相近，因为等值元素被一次性排除

### 7.3 总结

> **QuickSelect 在实际中更快（常数小），LinearSelect 提供了理论上的最坏情况保证但付出了常数代价；增大 Q 可以显著缩小两者差距。**
