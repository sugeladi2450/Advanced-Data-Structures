## 读者-写者同步策略设计报告

### 1. 问题核心

3 个线程（Reader-1、Reader-2、Writer）循环执行 N 轮，每轮必须满足：**两个读者都读完 → 写者写入 → 下一轮开始**。关键约束是同一轮两个读者读到的值必须相同。

### 2. 同步策略：三轮信号量接力

采用 **3 个信号量** 构成接力链：

```
readerDone (初始0) ──→ Writer 等待读者
writerDoneForReader1 (初始0) ──→ Reader-1 等待写者
writerDoneForReader2 (初始0) ──→ Reader-2 等待写者
```

再加 1 个互斥锁 `mutex` 保护共享计数器 `readCount`。

### 3. 每轮的同步流程

```
Reader-1                Reader-2                Writer
  │                       │                       │
  ├─ if(i>0) 等 writerDone1                       │
  ├─ 读 value, 打印       │                       │
  │                       ├─ if(i>0) 等 writerDone2
  │                       ├─ 读 value, 打印       │
  ├─ mutex: readCount++ (≠2, 不通知)              │
  │                       ├─ mutex: readCount++ (=2!)
  │                       ├─ 释放 readersDone ───→├─ 收到 readersDone
  │                       │                       ├─ 写入 value, 打印
  │                       │                       ├─ 释放 writerDone1 ──→├─ 收到
  │                       │                       ├─ 释放 writerDone2 ──→├─ 收到
  │                       │                       │                   (下一轮)
```


### 4. 正确性保证

- **同一轮两个读者读到相同值**：写者在两个读者都读完前被 `readersDone` 阻塞，无法修改 `value`
- **写者写入时无读者读取**：写者释放 `writerDone` 后才允许读者进入下一轮
- **无死锁**：每轮信号量释放和获取严格匹配——2 个读者各释放 1 次（第 2 个读者释放 `readersDone`），写者释放 2 次（`writerDone1` + `writerDone2`）
- **第一轮读者直接读**： `if (i > 0)` 条件让第 0 轮跳过等待写者的信号量
- **写者等两个读者**：`readersDone` 初值为 0，写者先 `acquire()` 阻塞；第二个读者完成后 `release()` 唤醒
- **读者等写者**：写者释放 `writerDone1` 和 `writerDone2`，各读者独享一个信号量，杜绝一个读者抢走两个许可
- **保护 readCount 竞争**： `mutex` 互斥锁包裹 `readCount` 的增减和判断
